# Research Basis

This skill applies requirements engineering principles:
- separate explicit, inferred, and assumed requirements
- keep functional and non-functional requirements separate
- make requirements testable and traceable
- use acceptance criteria for validation
- analyze dependencies and consequences before implementation
- separate problem-space analysis from solution-space coding

NFR categories align with common software quality models:
functional suitability, performance, compatibility, usability/accessibility, reliability, security/privacy, maintainability, portability, observability/operability, compliance, retention.
