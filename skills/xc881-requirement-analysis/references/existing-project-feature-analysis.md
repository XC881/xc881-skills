# Existing Project Feature Analysis

Use for new features or product-impacting changes in an existing project.

Include:
- current behavior
- affected user flows
- affected UI/API/domain/data/permissions/integrations/tests
- must-not-regress behavior
- compatibility and migration impact
- rollout needs
- regression test targets
- minimal safe implementation order

If repository files are available, inspect them before project-specific claims. Otherwise label assumptions.
