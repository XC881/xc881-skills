# English Spec First Policy

Use before requirement analysis when the request is rough, multilingual, mixed Chinese-English, ambiguous, emotional, or scattered.

Output:

```text
Standard English Spec:
Goal:
Inputs:
Constraints:
Output:
Process Notes:
```

Rules:
- Compact only.
- Preserve names, paths, APIs, technologies, metrics, deadlines exactly.
- Separate hard requirements from inferred preferences.
- Do not invent requirements.
- Ask one blocking question only when needed.
- Continue to requirement analysis after normalization.
