# Source Priority Policy

Use when web research is performed.

## Principle / algorithm / model / protocol

1. Survey paper or benchmark paper
2. Original paper or arXiv preprint
3. Standard/RFC/specification
4. Official docs
5. Maintainer notes

## Technology choice / implementation

1. Official docs
2. Release notes/changelog
3. Standards/RFCs
4. Maintainer GitHub issues/PRs/discussions
5. Benchmark repos and reproducible examples
6. High-quality developer forum answers
7. Blogs only as support

## Security / dependency risk

1. Vendor/security advisory
2. CVE/NVD, GHSA, OSV
3. CWE/OWASP
4. Maintainer issue/PR/release note
5. Incident reports

Rules: prefer primary sources, match exact version/context, cite sources, rank conflicts by authority/recency, never cite irrelevant sources.
